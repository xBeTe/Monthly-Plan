# 月末总结与下月计划生成指令 (Summarize Plan)

在月末最后一天触发此指令，对当月计划进行盘点并自动生成下月计划。

## 步骤 1：读取并解析当月计划数据
- 读取当月 JSON 计划文件（`monthly_plan_YYYY-MM.json`）。
- 提取所有任务及其子任务数据。

## 步骤 2：计算完成率与延期任务
- **计算完成率**：
  - 统计完成的任务数量（`status` 为 `Done` 或 `Reviewed`）。
  - 统计总任务数量（包括子任务）。
  - 完成率 = (完成任务数 / 总任务数) × 100%
- **识别延期任务**：
  - 筛选出 `status` 不为 `Done` 或 `Reviewed` 的任务。
  - 检查这些任务的 `end_date_planned` 是否已过当前月份。
  - 将所有已过截止日期的未完成任务标记为“延期任务”。

## 步骤 3：生成客观点评总结
- 基于完成率和延期任务情况，生成一段客观的点评总结。
- 总结应包含：
  - 本月完成率（百分比）。
  - 关键成就（已完成的高优先级或重要任务）。
  - 存在的问题（延期任务的数量和原因分析，如假期影响、资源不足等）。
  - 整体评价（例如：完成情况良好、面临挑战、需改进等）。

## 步骤 4：自动生成下月计划框架
- 根据当前月份（`YYYY-MM`），自动确定下个月（`YYYY-MM+1`）。
- 创建新的 JSON 计划文件（`monthly_plan_YYYY-MM+1.json`）的基本结构。
- **自动继承延期任务**：
  - 将步骤 2 中识别出的所有“延期任务”及其子任务，自动复制到下个月的计划中。
  - **重要调整**：
    - 将延期任务的 `status` 重置为 `Todo`。
    - 将 `start_date_planned` 和 `end_date_planned` 调整为下个月的合理时间范围（例如：下月第一天至最后一天，或根据实际情况顺延）。
    - 清空 `start_date_actual` 和 `end_date_actual`。
    - 保留 `description`、`category`、`priority`、`severity` 等原有属性。

## 步骤 5：输出结果
- 输出本月计划的**完成情况总结**。
- 列出**延期任务清单**，并询问用户是否同意将其顺延至下月计划。
- 告知用户下月计划已自动生成，并附上新月的 JSON 文件路径。
- 提示用户在下月初对新生成的计划进行审查和调整。
