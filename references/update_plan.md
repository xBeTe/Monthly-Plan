# 更新计划与日常提醒指令 (Update Plan)

## 定时任务触发
- **每周一提醒**：触发定时任务后，主动向用户推送当前的【月度看板】，并发消息：“新的一周开始了，请根据上周执行情况刷新当前任务进度，或者增删任务”。
- **每天提醒**：触发定时任务时，主动读取当月 JSON 文件，筛选出状态为“In Progress”或按日期落在当天的任务，用友好的早安问候，整理并推送给用户“今日待执行计划”。

## 用户主动更新处理
- **收到更新诉求**：首先获取当前日期，利用**核心规则2（时间感知）**读取对应月份（如 `monthly_plan_2026-03.json`）的 JSON 文件。
- **解析更新意图**：分析用户是想修改状态、进度、增加新任务或是删除旧任务。
- **Schema校验与默认值**：在更新 JSON 之前，严格对照 `references/schema.md` 中的规范进行校验。
  - 对于**新增任务/子任务**：必须引导用户补全全部的**非空字段**（如 `pic` 负责人、`category` 等），并自动为其应用默认值（如 `progress` 默认为 0%，计划起止时间默认为当月首尾。ID以特定格式自动生成）。
  - 对于**修改操作**：确保修改后的枚举值（例如 `status` 只能是 Schema 中允许的 Todo、In Progress、Done、Reviewed、Blocked 之一）依然符合要求。
- **预览确认写入**：遵循**核心规则4（操作确认）**，针对用户的更新，先呈现一份“更新前 vs 更新后”的差异或总结预览清单。询问：“请确认这些修改无误，确认后我为您保存。”
- **反馈修改成功**：收到用户确认无误后，写入 JSON，并以简洁的确认话术反馈修改成功的列表。
