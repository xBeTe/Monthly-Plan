# 生成看板指令 (View Board)

按以下步骤生成包含任务进度与法定假期标注的看板视图。

## 步骤 1：读取计划数据
- 读取当月 JSON 计划文件（`monthly_plan_YYYY-MM.json`），提取所有任务和子任务。

## 步骤 2：查询当月法定假期
- 读取 `user_profile.json` 获取用户的 `country` 和 `region`。
  - 若 `user_profile.json` 不存在或 `country` 为空，遵循**核心规则5**，先向用户询问国家和地区并保存后再继续。
- 通过**联网搜索**该国家/地区官方渠道公布的当年法定节假日，提取**当月**所有假期的名称与日期范围（格式：`YYYY-MM-DD` 至 `YYYY-MM-DD`）。

## 步骤 3：构建看板视图

### 3-A. 假期卡片（置顶显示）
- 在看板顶部创建名为 `🎉 法定假期` 的区域。
- 为当月每个法定假期创建一个卡片，包含：
  - 假期名称
  - 日期范围
  - 醒目标识（如使用红色边框或背景）
- 若当月无假期，则不显示此区域。

### 3-B. 任务卡片（按状态分组）
- 按照 `status` 字段（Todo、In Progress、Done、Reviewed、Blocked）对任务进行分类汇总。
- 为每个任务创建一个卡片，包含：
  - **任务 ID**（如 T01）
  - **任务名称**（在卡片标题中显示）
  - **进度条**：根据 `progress` 字段生成可视化进度条（如 `[██░░░]`）。
  - **优先级**：高、中、低
  - **重要程度**：Minor、Major、Cruci、Crucial
  - **截止日期**：`end_date_planned`（若落在假期内，需特别标注）
  - **负责人**：`pic`
  - **子任务**：在卡片内以缩进列表形式展示，包含子任务 ID、名称、负责人和状态。

### 3-C. 排序规则
- **组内排序**：
  - 优先按“高、中、低”优先级排序
  - 其次按“Minor、Major、Cruci、Crucial”重要程度排序
  - 最后按任务创建时间（或 ID 顺序）
- **跨组排序**：
  - 假期卡片始终置顶
  - 任务卡片按状态分组显示（Todo → In Progress → Done → Reviewed → Blocked）

## 步骤 4：输出结果
- 以 Markdown 格式清晰地展示所有任务卡片和假期卡片。
- 确保卡片布局美观易读，并附带简短的统计信息（如各状态任务数量、完成率）。
- 假期卡片应置于任务卡片之前，并有醒目标识。
